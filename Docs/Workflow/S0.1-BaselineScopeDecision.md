# Baseline Scope Decision Record

- **Title:** Baseline Scope Decision Record
- **Scope:** This decision record establishes the approved baseline scope and required output set for Stage 0.1 of the AxionFrame roadmap, aligned to the current project documentation and the toolchain versions declared by the repository.

## Purpose

This document is the Stage 0.1 deliverable defined in `roadmap.md`. It confirms the product scope for the AxionFrame **Build** and **Final Output** flows and records the manufacturing handoff expectations those flows must satisfy.

This baseline defines the intended product scope for roadmap execution. It does not claim that all scoped behavior is already implemented in the current codebase.

## Stage 0.1 Decision

AxionFrame baseline scope is confirmed as a SolidWorks add-in that supports a guided workflow for generating a modular, height-adjustable X-frame table and producing manufacturing deliverables from that generated design.

The baseline is split into two user-visible operations:

1. **Build**
2. **Final Output**

Both operations are mandatory parts of the product definition for roadmap planning, architecture, validation, and release.

## Dependency Baseline Notes

- The repository does not currently declare NuGet package dependencies.
- SolidWorks interop assemblies are referenced from `$(SolidWorksInstallDir)` in `API/Addin/AxionFrame.csproj`.
- Because the repository does not pin separate DLL version numbers for those assemblies, this baseline ties the API dependency set to the installed SolidWorks 2026 environment required by the project.

## Confirmed Build Scope

The **Build** operation is confirmed to include the following in-scope behavior:

1. Parameter loading.
2. Parameter validation.
3. CAD part generation.
4. CAD assembly generation.
5. Configuration activation for target table heights.

## Build Scope Interpretation

For Stage 0 planning and all downstream work:

- Parameter input is part of the product surface and must be represented in add-in workflows.
- Validation is mandatory before geometry generation.
- CAD generation includes both parts and assemblies, not parts alone.
- Height-adjustable behavior is core product scope, not an optional extension.
- Build completion is expected to leave a reviewable CAD result inside SolidWorks.

## Confirmed Final Output Scope

The **Final Output** operation is confirmed to include the following in-scope deliverables:

1. STEP export.
2. DXF export for plate components.
3. BOM generation.
4. Validation report generation.
5. Output packaging in run-based folders under `Output/`.

## Required Baseline Output Set

The minimum required output set for the product baseline is:

- Assembly files
- Part files
- STEP exports
- DXF exports for plate components
- BOM outputs
- Validation reports

## Mechanical Handoff Confirmation

The baseline output set satisfies the documented manufacturing handoff requirement because it covers:

- Model definition for assemblies and individual parts
- Neutral-format exchange through STEP
- Fabrication drawings for plate components through DXF
- Purchasing and fabrication traceability through BOM output
- Quality and release traceability through validation reports

## Core Mechanical Scope Areas

The baseline product scope must support the following mechanical domains:

- Frame member layout and profile selection
- Pivot joint geometry and hole strategy
- Height indexing strategy for supported table configurations
- Plate and brace details with manufacturable dimensions
