# S2.3 Deterministic Naming Rules Feature Ticket

- **Title:** S2.3 Deterministic Naming Rules Feature Ticket
- **Scope:** This document records the Definition of Ready feature ticket and completion evidence for roadmap step `S2.3`, covering deterministic naming grammar implementation, normalization rules, domain-token application, stable naming hooks, and naming verification expectations for the AxionFrame baseline.

## Feature DoR Record

- **Feature ID:** `AXF-S2.3-NAMING-RULES`
- **Feature Title:** Deterministic naming grammar and normalization implementation
- **Roadmap Step:** `S2.3`
- **Owner:** SW+ME
- **Primary Domain:** `Frame`
- **Supporting Core Service Areas:** `Validation`, `Infrastructure`
- **Implementation Scope Summary:** Implement the documented naming grammar and normalization rules so downstream generation modules can produce deterministic feature, mate, configuration, export, and validation identifiers. #### Configuration Readiness

- **Configuration Keys:** `frame.naming.ruleSet`, `pivot.naming.mates`, `height.supportedConfigurations.values`, `height.indexing.activation.strictDeterminism`, `plateBrace.naming.ruleSet`
- **Schema/Placeholder Status:** `Defined`
- **Default Value Notes:** Naming rule-set inputs are already defined in the configuration schema and currently resolve to `AXF_STANDARD_V1`. Supported height values are already standardized in Stage 2.2 and should drive deterministic configuration-token generation without decimal suffixes.
- **Range / Limit / Tolerance Notes:** Naming conformance is an exact string-match rule with allowed deviation `0`. Names must use uppercase ASCII letters, digits, and underscore separators only; must not emit duplicate underscores; must not append timestamps or random identifiers; and must normalize approved discrete numeric states to decimal-free tokens such as `680`.

#### Output Expectations

- **Expected CAD Outputs:** Deterministic frame feature names, pivot mate names, supported-height configuration names, and plate/brace feature names aligned to the canonical grammar.
- **Expected Final Outputs:** Deterministic export artifact identifiers and validation section identifiers aligned to the documented prefix and grammar rules.
- **Expected Report Sections / Trace Points:** `traceability.naming.frame`, `traceability.naming.pivot`, `height.activation`, `traceability.naming.plateBrace`, `AXF_EXP_DXF_PLATE_SET`, `AXF_VAL_PLT_TRACEABILITY`

#### Planned Tests

- **Unit Tests Planned:** Prefix selection; domain-token mapping; descriptor normalization; duplicate-underscore collapse; uppercase conversion; decimal-free height token generation; repeat-run stable naming for identical inputs.
- **Integration Tests Planned:** Naming application through downstream Build orchestration surfaces for frame features, pivot mates, supported-height configurations, and plate/brace traceability identifiers.
- **Regression Checks Planned:** Re-run identical naming inputs and confirm identical feature, mate, configuration, export, and validation identifiers across repeated executions.
- **Manual Verification Planned:** Compare generated name samples against `Docs/Software/DeterministicNamingStandard.md` and the naming-hook rows in `Docs/Mechanical/S1.1-RequirementTraceMatrix.md`.
