# Roadmap Tracking Board and Evidence Registry

- **Title:** Roadmap Tracking Board and Evidence Registry
- **Scope:** This document establishes the Stage 0.2 operational tracking board for AxionFrame and defines the evidence registry fields required to track roadmap progress, verification, and cross-functional review.

## Purpose

This document is the Stage 0.2 deliverable defined in `roadmap.md`. It provides:

- The standard board columns for roadmap execution
- The required evidence registry schema
- The manufacturability review fields required for mechanical validation
- A preloaded board entry for every roadmap step with explicit dependency links

This board is the repository bootstrap artifact for roadmap tracking. It can be mirrored into an external project board without changing the field model defined here.

## Stage 0.2 Decision

AxionFrame uses a single standardized tracking model for software and mechanical delivery. Every roadmap step is tracked through a consistent workflow state and must carry evidence fields that satisfy the roadmap completion standard in `roadmap.md`.

## Board Workflow Columns

Every roadmap step must exist in exactly one of the following board columns:

1. **Not Started** — work has not begun and no execution evidence exists yet
2. **In Progress** — implementation or review work is actively underway
3. **Blocked** — work cannot proceed because a dependency, environment, or approval is missing
4. **In Review** — deliverables exist and are awaiting SW, ME, or joint review
5. **Done** — deliverable, verification evidence, and acceptance metric are all complete

## Required Evidence Registry Fields

Each board item must carry the following mandatory evidence fields:

- **Step ID**
- **Artifact Path/URL**
- **Verification Evidence Link**
- **Acceptance Metric Result**
- **Reviewer(s)**

## Required Mechanical Review Fields

To satisfy the Stage 0.2 mechanical task, each board item must also expose manufacturability review fields:

- **Manufacturability Review Required** — `Yes` or `No`
- **Manufacturability Reviewer**
- **Manufacturability Evidence**
- **Manufacturability Status** — `Not Required`, `Pending`, `Approved`, or `Changes Required`

## Field Definitions

| Field | Required | Purpose |
|---|---|---|
| Board Column | Yes | Current workflow state |
| Step ID | Yes | Unique roadmap step identifier |
| Stage | Yes | Stage grouping for gate tracking |
| Step | Yes | Step title exactly aligned to `roadmap.md` |
| Owner | Yes | Accountable role or lead role |
| Complexity | Yes | `L`, `M`, or `H` from the roadmap |
| Dependencies | Yes | Upstream prerequisites exactly aligned to `roadmap.md` |
| Artifact Path/URL | Yes | Primary deliverable location |
| Verification Evidence Link | Yes | Link or path to proof of completion |
| Acceptance Metric Result | Yes | Explicit pass/fail or completion result |
| Reviewer(s) | Yes | SW, ME, or joint reviewers |
| Manufacturability Review Required | Yes | Whether ME review is mandatory |
| Manufacturability Reviewer | Yes | Assigned ME reviewer or `N/A` |
| Manufacturability Evidence | Yes | Link/path to manufacturability evidence or `N/A` |
| Manufacturability Status | Yes | Current manufacturability review state |
| Notes | No | Supplemental execution notes |

## Board Operating Rules

- A step cannot move to **Done** unless the deliverable artifact exists.
- A step cannot move to **Done** unless verification evidence exists.
- A step cannot move to **Done** unless the acceptance metric is explicitly recorded.
- A step with manufacturing impact should not move to **Done** until manufacturability status is `Approved`.
- Dependency fields must be preserved exactly so gate readiness is auditable.
- When mirrored to an external tracker, field names should remain unchanged.
