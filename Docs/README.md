# AxionFrame Documentation

This documentation hub provides a clear, scalable structure for software engineering, mechanical engineering, project workflow, and architectural decisions.

## Structure

- **Software/**: Software architecture, coding patterns, and implementation guidance.
- **Mechanical/**: Mechanical design intent, manufacturing constraints, and engineering references.
- **Workflow/**: End-to-end operational flow for development, generation, validation, and release.
- **Governance/**: Documentation standards, naming, versioning, and ownership practices.
- **ADR/**: Architecture Decision Records for important technical choices.

## Recommended Reading Order

1. `Workflow/Project-Lifecycle.md`
2. `Software/Architecture.md`
3. `Mechanical/Design-and-Manufacturing.md`
4. `Governance/Documentation-Standards.md`

## Ownership Model

- Software Lead owns `Software/`.
- Mechanical Lead owns `Mechanical/`.
- Tech Lead owns `Workflow/` and `ADR/`.
- Project Lead owns `Governance/`.

## Change Policy

Every documentation update includes:

- A clear purpose statement.
- Version and date in the document header.
- Links to related documents when applicable.
