# Documentation Standards

- **Version:** 1.0
- **Last Updated:** 2026-02-24

## Naming Standard

- Folder names use PascalCase.
- Markdown file names use PascalCase with descriptive scope.
- Versioned documents include a version header.

## Document Header Standard

Each document includes:

- Title
- Version
- Last Updated date
- Scope

## Content Quality Standard

Each document provides:

- Purpose and scope.
- Clear structure with headings.
- Actionable lists and explicit outputs.
- Links to related documents.

## Update Workflow

1. Update content.
2. Confirm links and references.
3. Record structural decisions in ADR when relevant.
4. Commit with a clear documentation message.
